# 作品经验记录

每完成一个作品都要在本文件追加项目记录。工具组合只代表当次项目经验，不是后续项目的硬性要求。

项目记录应包含：日期、类型、有无台本、输出范围、关键工具/后端、主要 QC 发现、可复用经验、已同步到 `references/style.md` / `references/terms.md` / `references/risk-notes.md` / `data/subtitle_risk_patterns.json` 的条目，以及仍在 `references/pending.md` 的待验证问题。

如果某个作品没有发现新的通用风格、术语或易错词，也要记录“无新增可泛化规则”，这样后续 agent 能知道这个项目已经完成学习回收。

## RJ01201653

- 日期：2026-06-12
- 类型：有初稿台本；本篇、EX、促销均生成独立 SRT
- 本次使用过的工具组合：`mlx-community/whisper-large-v3-mlx-4bit` + `Qwen3.6-27B-MLX-VL-oQ6`。这是项目记录，不是固定要求。
- 关键经验：
  - 台本用于纠错和确认剧情，不直接作为字幕时间轴来源。
  - 促销部分必须单独校对，不能假设本篇修正会覆盖促销。
  - `しっこ/シコシコ` 在 ASR 中极易误成 `尿尿/嘘嘘`。
  - `オナサポ` 与 `フェラ` 必须区分。
  - 长段拟声、亲吻、舔耳保留氛围即可，不必逐音节翻译。

## RJ01533156

- 日期：2026-06-12
- 类型：无台本；本篇与短轨均生成独立 SRT
- 本次使用过的工具组合：`mlx-community/whisper-large-v3-mlx-4bit` + oMLX `Qwen3.6-27B-MLX-VL-oQ6`。
- 关键经验：
  - 本作核心词 `早漏（そうろう）` 的 ASR 错词非常集中，必须整体扫描 `僧侶/騒動/ソウロウ/Souro` 及其中文误译。
  - oMLX 跑 27B 模型时，默认内存保护可能不足；本次使用 `--memory-guard-gb 30`。
  - 批量翻译应使用进度条和断点文件；正常推进时不需要频繁文字汇报。
  - 无台本结尾的 `ご視聴ありがとうございました` 应优先视为 ASR 幻觉候选。
  - 模型质检应设为必经步骤。本次二次 QC 找出了初翻和关键词扫描未覆盖的逻辑错位，例如 `射精` 误链到 `写真/书生/照片/姿势`，以及 `オナニー` 误链到 `オーナー/主人/服侍`。
  - QC 建议不可照单全收。本次自动建议中也出现过生硬替换，需要人工回看相邻字幕后再决定是否应用。
