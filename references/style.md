# ASMR 中文字幕风格与可读性规则

用于翻译、手修、可读性判断和最终交付前自检。

## 总体中文风格

- 目标是“能挂字幕、读得顺、符合情境”，不是论文式逐字翻译。
- 亲密、撒娇、挑逗、犹豫、停顿要保留，但中文要自然。
- 对角色语气要统一：同一角色的敬语、调侃、害羞、强势程度不能前后乱跳。
- 同一句里不要混太多英文、日文和中文；必要术语如 `coser`、`cos`、`M 男` 可以保留。
- 长段重复音效不要逐字铺满字幕，可简化为短拟声或方括号提示。
- 不要把 ASR 幻觉翻得很认真；明显荒唐时要回看日文、台本或上下文。
- 有台本项目以台本为主要参考。模型质检只做辅助补漏；当模型建议与台本、相邻剧情或角色设定冲突时，优先相信台本和上下文。
- 初翻完成后必须做模型质检。尤其是无台本项目，很多问题不是孤立错词，而是 ASR 同音误识别带来的整段逻辑错位；单靠关键词扫描和人工粗读不够稳。
- 模型质检的结果只能作为候选。每一项都要以 `accept`、`reject` 或 `defer` 闭环：接受需要来源证据和最小替换文本，误报需要拒绝理由，证据不足需要写明人工或音频复核方式。
- 语义、人称、术语、动作、情绪、因果和程度改动不能只凭 QC 建议接受。台本优先，其次音频确认，再其次足够的日文 ASR 与相邻语境；无法证明时进入复核队列。
- 修正采用最小充分修改：只解决确认的问题，不为“更自然”重写整句，更不能补入原文没有的人物、动作、情绪、因果或程度。

## SRT/VTT 书写规范

- 不改时间轴，不改编号，除非用户明确要求重切时间轴。
- 每条字幕应适合 ASMR 的放松阅读状态。默认把 10 个中文字符/秒作为可读性 warning 阈值；这不是硬性错误，而是提醒人工回看。
- 不为了降低阅读速度而把字幕切得太碎。稍长但语义完整、停留时间够、读起来顺的字幕可以接受；连续很多条极短字幕频繁闪现也会破坏 ASMR 沉浸感。
- 一条里可用换行，但不要让单行太长；优先保持语义自然和听感稳定。
- 默认最终输出 `.zh.vtt`；用户可选择 `srt` 或 `both`。`.zh.srt` 仍作为工作中间稿。
- 成品目录只放该音频目录对应的最终字幕，不混入 ASR、partial、QC/report 或 review notes。

## 拟声与音效

- 校对拟声词时要对照日文台本或日文 ASR，先判断音效类型，再压缩成中文提示。不要只按中文重复字符猜成 `[亲吻声]`、`[摩擦声]` 等。
- 长段无意义 ASR 幻觉，如“感谢观看/请订阅”，如果实际是音效，改成对应音效提示。
- 如果拟声中夹着有效台词，只压缩纯音效部分，台词必须保留。
- 可简化为：
  - `[亲吻声]`
  - `[舔耳声]`
  - `[亲吻声和舔耳声]`
  - `[长长的喘息声、亲吻声、舔耳声]`
  - `撸撸……`
  - `啪啪、啪啪……`

## 常见语气处理

### 女主对“先生”的称呼

- 来源：`RJ01201653` / 全篇 / 有台本 / 2026-06-12
- 推荐译法：保留 `老师`
- 说明：不要自动改成“你”或“先生”。可根据句子自然度混用“老师/你”，但角色称呼要稳定。
- 标签：人称 / 语气

### 撒娇式命令

- 来源：`RJ01201653` / 02、07、促销03、10 / 有台本 / 2026-06-12
- 推荐译法：`来，快说`、`要好好配合我的声音哦`、`别一下子就不行哦`
- 说明：既有命令感，也要保留亲密关系，不要翻成冷冰冰命令句。
- 标签：语气

### 羞耻台词复述

- 来源：`RJ01201653` / 02、促销03、10 / 有台本 / 2026-06-12
- 规则：女主让男方说的话要加引号或明确人称。
- 推荐译法：`“求求你，帮我撸撸小弟弟嘛”`
- 常见错误：把男方要复述的话译成女主自己在说。
- 标签：人称 / 易错

### 喘息和停顿

- 来源：`RJ01201653` / 全篇 / 有台本 / 2026-06-12
- 推荐译法：`哈啊……`、`嗯……`、`呼……`
- 规则：保留少量情绪，不要每个喘息都铺满屏幕。
- 标签：拟声 / 语气
