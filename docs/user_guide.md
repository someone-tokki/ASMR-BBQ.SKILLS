# ASMR Subtitle Translator 用户使用指南

这份文档面向使用 Skill 的用户。你不需要记住脚本名；多数时候只要把作品路径和你的要求告诉 agent，它会按 Skill 流程分流、调用本地模型、生成字幕、QC、导出和更新学习库。

## 可以怎么开始

最简单的说法：

```text
用 asmr-subtitle-translator 处理这个作品：/path/to/RJxxxxxx
```

如果你已经有日文 ASR 或中文字幕，也可以直接说明：

```text
这个作品已经有 .ja.asr.srt，请复用已有 ASR，继续翻译和 QC。
```

```text
我已经有 .zh.srt，帮我跑结构检查、风险扫描、可读性和模型 QC，然后修明确问题。
```

## 你可以指定翻译范围

开跑前，Skill 会先列出作品文件夹下发现的音频文件夹，让你确认本轮要翻译哪些文件夹或具体音频。试听、DLC、EX/free talk、bonus、特典不会被默认歧视或跳过；但本轮范围必须由你确认后才会开跑。

你可以这样指定范围：

```text
只翻本篇，不翻试听和特典。
```

```text
这次只处理 track01.wav 和 track02.wav。
```

```text
促销音频也要翻，和本篇一起导出。
```

```text
这次用有 SE 版本做 ASR，不要默认选无 SE。
```

如果文件夹里有无 SE 版本，默认会先确认它能和普通音频按轨道对应，再用可对齐的无 SE 音频做 ASR，因为识别通常更干净。同一轨道无 SE 同时有 MP3/WAV 时，默认优先 MP3 以加快识别；如果报告提示试听/裁剪/占位或映射不清，会先提醒核对。用户明确指定版本时，以用户指定为准。

你也可以直接按文件夹选择：

```text
先列出音频文件夹，这次只翻本編和EX，不翻体験版。
```

```text
这次全部文件夹都烤，包括试听和特典。
```

## 开跑前会确认什么

正式 ASR、翻译、QC 前，Skill 会做 Preflight，确认：

- 本轮翻译哪些音频文件夹或具体文件。
- 质量模式：`draft`、`standard`、`premium` 或 `polish`。
- ASR、翻译、QC 分别用什么 backend/base URL/model。
- 是否复用已有 `.ja.asr.srt`。
- 最终输出 `vtt`、`srt` 还是 `both`。

确认后会写入 `subtitle_project/run_profile.json`。如果这个文件没有确认，模型调用会被 `check_preflight.py` 拦住，避免 agent 自己脑补开跑。

注意：auto mode 不等于你已经确认。除非你明确说“全部按默认跑”“你决定”“不用问”，否则 agent 应该先把这些选择列出来等你确认。正式 ASR、翻译、QC 脚本在收到 `--project-root` 时也会检查这个确认记录；如果缺少确认来源或音频范围依据，会直接拒绝开跑。

常用质量模式：

- `draft`：速度优先，适合先出粗稿。
- `standard`：日常默认，dynamic chunk + two-pass QC。
- `premium`：更慢，更小 chunk，更强 QC。
- `polish`：复用已有 ASR/翻译，只做后续精修。

## 你可以指定输出格式

默认最终输出是 `.zh.vtt`，工作中间稿仍是 `.zh.srt`。你可以明确要求：

```text
最终只要 srt。
```

```text
最终导出 vtt 和 srt 两种。
```

最终字幕默认放在作品文件夹下的 `subtitles/`，工程文件和中间文件放在 `subtitle_project/`，避免散落在作品根目录。

## 你可以指定模型和后端

Skill 会尽量使用本地或项目配置的模型服务。ASR、翻译、QC 是三个不同工况，可以使用不同模型。

你可以这样说：

```text
ASR 用本地 Whisper，翻译和 QC 用 Ollama 的 qwen3.6-27b。
```

```text
翻译走 LM Studio，端口是 1234，QC 也用同一个模型。
```

```text
ASR 走 local-asr-api，翻译和 QC 走 oMLX。
```

重要规则：模型 QC 必须调用配置的本地/项目 QC 模型接口，不能让 agent 用自己的模型偷偷替代。第二轮、第三轮 QC 精修也一样。如果本地 QC 服务不可用，agent 应该停下说明，而不是自审。

当翻译和 QC 使用不同模型时，Skill 会在进入 QC 前做一次模型阶段检查。这个检查会确认本地服务是否已经能调用 QC 模型；如果遇到 HTTP 500，常见原因是翻译模型还占着显存/内存，导致 QC 模型加载失败，或本地后端不能自动热切换。此时 agent 应提示你在 oMLX/LM Studio/Ollama 等后端释放上一模型、切到 QC 模型或重启服务，然后再继续。

## QC 不只是“跑一遍检查”

翻译完成后，Skill 默认会做一轮强制模型 QC。模型 QC 的输出只是候选建议，不会直接自动改字幕；agent 会结合日文 ASR、台本、上下文、风险库和语料规则判断哪些该接受。

如果第一轮 QC 后你仍然觉得台词不满意，可以要求追加 QC 精修：

```text
再做一轮 QC 精修，自动判断这部作品的主题和语气，重点看台词是否太硬。
```

```text
做 guided QC：现在的翻译太书面，希望更像耳边低语，但不要过度改写。
```

```text
再跑一轮 QC，重点检查称呼、亲密感、成人术语是否前后一致。
```

追加 QC 有两种常用模式：

- `auto`：agent 先根据作品标题、DLsite 信息、轨道名、ASR 和当前字幕自动生成情境画像，再让本地 QC 模型做聚焦检查。
- `guided`：你提供风格说明、问题描述或引导词，agent 把你的要求写入本轮 QC 上下文。

你也可以直接对 `qc_report.json` 提意见：

```text
根据 qc_report.json，我觉得第 35 条和第 82 条建议可以接受，其他先别动。
```

```text
我觉得目前整体太冷淡，请基于 QC 报告再做一轮更亲密口语的修正建议。
```

## 你可以要求只读检查

如果你暂时不想修改字幕，可以让 agent 只生成报告：

```text
只检查，不修改：跑结构校验、风险扫描和可读性检查。
```

```text
只做高风险词和 ASR 错词扫描，先不要改文件。
```

```text
帮我看这份字幕有没有太快、太长、切得太碎的问题。
```

常见检查包括：

- 结构校验：编号、时间轴、空字幕、重叠、ASR/ZH 是否成对一致。
- 风险扫描：成人术语、ASR 易错词、翻译高风险候选。
- ASMR 可读性：默认按较慢阅读速度检查高 CPS、过长行、过多行、过碎闪现。
- 模型 QC：调用配置的本地 QC 模型，检查明显错译、上下文错位和不自然字幕。

如果 Whisper ASR 只出现很小的时间重叠，你也可以让 agent 在翻译前先做一次保守的 ASR 时间轴修复：

```text
如果只是 Whisper 的轻微时间重叠，就先自动修小 overlap；大 overlap 先报告，不要乱改。
```

这个修复只针对 `.ja.asr.srt` 的 ASR 时间轴问题，不属于普通翻译或 QC 自动修改。若已经有对应 `.zh.srt`，默认不会只改 JA 而不提醒你。

## 你可以提供风格偏好

ASMR 字幕不是普通影视字幕。你可以在开工前给风格要求：

```text
翻译要自然简中，亲密但不要油腻。
```

```text
称呼统一用“你”，不要用“您”。
```

```text
喘息和亲吻声不要逐字堆重复，长段可以压缩成可读提示。
```

```text
保留一点拟声节奏，但不要让字幕刷屏。
```

```text
这部是姐姐系角色，语气温柔成熟，不要太少女。
```

如果你没有给风格，Skill 会读取语料库和项目经验，尽量使用自然、亲密、可读的简中 ASMR 字幕风格。

## 你可以让 Skill 学习本次项目经验

每完成一个作品，Skill 会尝试从翻译、QC、风险扫描、可读性检查和人工修正里提炼经验，写入学习库。你可以主动要求它总结：

```text
这次结束后，把可复用的术语、风格经验和易错词更新到学习库。
```

```text
把这部作品里确认过的角色称呼和成人术语沉淀下来，下次同社团作品复用。
```

学习库通常会分成：

- 项目经验：每个作品都记录。
- 风格规则：口吻、称呼、拟声、可读性偏好。
- 术语库：角色称呼、固定词、成人术语译法。
- 风险库：容易错的 ASR 词、误报说明、需要人工确认的候选。

不确定的经验会先进 pending，不会直接当成全局规则。

学习库维护的详细规则见 `docs/learning_library_guide.md`。如果你希望某条经验只对当前作品生效、先放 pending、或不要写入全局术语库，可以直接告诉 agent。

## 可以让 agent 抓取作品信息

如果路径或文件夹名里能识别 RJ 号，并且你允许联网，agent 可以抓取 DLsite 元信息作为低强度上下文：

```text
如果识别到 RJ 号，就先抓 DLsite 信息辅助判断角色和主题。
```

DLsite 标题、标签、社团和简介只能帮助理解题材，不会被当成台本，也不能覆盖 ASR 或官方文本证据。

## 常见高级请求

```text
这次速度优先，但不要牺牲明显质量，QC 用 two-pass。
```

```text
这次用 draft 先粗烤，输出 vtt。
```

```text
这次用 premium，翻译和 QC 都用 27b。
```

```text
长音频允许断点续跑，中途停了下次接着跑。
```

```text
长音频先准备 ASR cache 和分段恢复计划，但不要用激进 VAD。
```

```text
已有 ASR 不要重跑，先校验后继续翻译。
```

```text
翻译前请先列出将要处理的全部音频，确认没有漏掉试听、DLC 和特典。
```

```text
导出前确认 final subtitles 文件夹里只有最终 vtt/srt，不要混入报告和中间稿。
```

```text
如果本地模型端口不可用，先告诉我，不要换成 agent 自己的模型。
```

```text
03:12-03:28 这里左右两边好像同时在说话但漏字幕，请做双声道补漏 ASR。
```

```text
这部有双人左右耳语，请主 ASR 后只扫描疑似漏识别片段，不要全片左右声道重跑。
```

双声道补漏默认先做候选检测或生成左右声道候选片段和 review，不会直接覆盖主 ASR 或最终字幕。如果检测结果不确定，agent 会明确提醒你甄别：那一段可能是台词，也可能只是 ASMR 音效、BGM、喘息或摩擦声。

## 质量底线

无论你要求速度、缓存、动态 chunk、二轮 QC 还是局部重跑，都不能破坏这些底线：

- 保持 SRT 编号、顺序、开始时间、结束时间不变。
- 翻译输出必须覆盖所有目标编号。
- context halo 只能作为上下文，不能写进输出。
- QC 建议只能作为候选，不能直接自动改字幕。
- ASMR 语义、角色关系、动作连续性优先于机械压缩 chunk。
- 缓存不能复用到内容、模型、prompt/schema、参数或上下文不一致的 chunk。

## 一句话模板

你可以直接复制改写：

```text
用 asmr-subtitle-translator 处理 /path/to/RJxxxxxx。默认翻全部音频，包括特典和试听；最终导出 vtt。ASR 优先无 SE，翻译和 QC 用本地 qwen3.6-27b。翻译后做强制 QC 并修明确问题；如果台词太硬，再做一轮 guided QC，方向是更亲密自然但不改剧情。完成后更新学习库。
```
